# 什么是 MSM？

MSM (Mosdns Singbox Mihomo Manager) 是一个统一管理 **MosDNS**、**SingBox**、**Mihomo** 的可视化平台。

## 项目背景

在网络服务管理中，MosDNS、SingBox 和 Mihomo 是三个常用的核心组件：

- **MosDNS**: 强大的 DNS 服务器，支持分流、缓存等功能
- **SingBox**: 通用代理平台，支持多种协议
- **Mihomo**: Clash 内核的增强版本

传统的管理方式通常依赖复杂的脚本和命令行操作，对于普通用户来说门槛较高。MSM 的目标就是通过现代化的 Web 界面，让这些服务的管理变得简单直观。

## 核心理念

### 1. 统一管理

将三个独立的服务整合到一个平台中，提供统一的管理界面和操作流程。

### 2. 可视化操作

通过直观的 Web 界面替代命令行操作，降低使用门槛。

### 3. 配置即代码

支持在线编辑配置文件，提供语法高亮和实时验证。

### 4. 历史可追溯

自动保存每次配置变更，支持一键回滚到任意历史版本。

### 5. 实时监控

通过 WebSocket 实时推送服务状态和日志，及时发现问题。

- ✅ 实时查看服务状态

- ✅ 操作审计日志
- ✅ 登录安全控制

## 技术特点

### 前后端分离

- 后端提供 RESTful API 和 WebSocket 接口
- 前端独立部署，支持 SPA 单页应用
- 也支持单二进制部署，前端嵌入后端

### 高性能

- Golang 后端，高并发处理能力
- React 前端，流畅的用户体验
- SQLite 数据库，轻量级高效

### 安全可靠

- JWT 令牌认证
- 多级权限控制
- HTTPS 加密传输
- 操作审计日志

### 易于部署

- 单二进制文件部署
- Docker 容器化部署
- Systemd 服务管理
- 支持反向代理

## 适用场景

MSM 适合以下场景：

1. **个人用户**: 需要管理自己的 DNS 和代理服务
2. **小团队**: 多人共同管理网络服务
3. **运维人员**: 需要可视化管理工具
4. **学习研究**: 了解网络服务的配置和管理
